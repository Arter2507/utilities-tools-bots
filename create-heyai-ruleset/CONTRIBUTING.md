# Hướng dẫn đóng góp / Contributing to @heyai-rules/heyai-ruleset 🚀

Trước hết, cảm ơn bạn đã cân nhắc đóng góp cho `heyai-ruleset`! Chính những đóng góp của cộng đồng làm cho không gian nguồn mở trở thành một nơi tuyệt vời để học hỏi, truyền cảm hứng và sáng tạo.
(First off, thank you for considering contributing to `heyai-ruleset`! It's people like you that make the open source community such an amazing place to learn, inspire, and create.)

## 🌈 Quy tắc ứng xử / Code of Conduct
Dự án này và tất cả những người tham gia đều được điều chỉnh bởi [Quy tắc ứng xử](CODE_OF_CONDUCT.md). Bằng cách tham gia, bạn được kỳ vọng sẽ tuân thủ các quy tắc này.
(This project and everyone participating in it is governed by the [Code of Conduct](CODE_OF_CONDUCT.md). By participating, you are expected to uphold this code.)

## 🛠️ Làm thế nào để đóng góp? / How Can I Contribute?

### Báo cáo lỗi / Reporting Bugs
- Kiểm tra [Issues](https://github.com/Arter2507/create-heyai-ruleset/issues) để xem lỗi đã được báo cáo chưa.
  (Check the Issues to see if the bug has already been reported.)
- Nếu chưa, hãy tạo issue mới bằng mẫu **Bug report**.
  (If not, create a new issue using the Bug report template.)

### Đề xuất cải tiến / Suggesting Enhancements
- Kiểm tra [Issues](https://github.com/Arter2507/create-heyai-ruleset/issues) để xem đã có ai đề xuất chưa.
  (Check the Issues to see if someone else has already suggested it.)
- Nếu chưa, hãy tạo issue mới bằng mẫu **Feature request**.
  (If not, create a new issue using the Feature request template.)

### Pull Requests
1. **Fork** repository về tài khoản của bạn.
   (Fork the repository to your own account.)
2. **Clone** bản fork về máy cục bộ.
   (Clone your fork to your local machine.)
3. **Tạo branch mới** cho công việc của bạn (ví dụ: `feat/new-rule-kit`).
   (Create a new branch for your work.)
4. **Thực hiện thay đổi** và thêm test nếu có thể.
   (Implement your changes and add tests if applicable.)
5. **Đảm bảo các test đều vượt qua** (`npm test`).
   (Ensure all tests pass.)
6. **Commit thay đổi** sử dụng [Conventional Commits](https://www.conventionalcommits.org/).
   (Commit your changes using Conventional Commits.)
7. **Push** branch lên bản fork của bạn.
   (Push your branch to your fork.)
8. **Gửi Pull Request** vào branch `main` của repository gốc.
   (Submit a Pull Request against the main branch of the original repository.)

## 🤝 Tiêu chuẩn / Standards
- **Conventional Commits**: Mỗi commit phải tuân thủ `feat:`, `fix:`, `docs:`, v.v.
  (Every commit must follow feat:, fix:, docs:, etc.)
- **Hỗ trợ song ngữ (Bilingual Support)**: Tất cả tài liệu phải được cung cấp bằng cả **Tiếng Việt** và **Tiếng Anh**.
  (All documentation must be provided in both Vietnamese and English.)
- **Bảo mật (Security)**: Tuyệt đối không commit bí mật hoặc dữ liệu nhạy cảm. Mọi đóng góp đều được quét mã độc.
  (Never commit secrets or sensitive data. All contributions are scanned for malware and backdoors.)

## 📝 Quy trình đánh giá PR / PR Review Process
- Mọi PR đều cần ít nhất một sự phê duyệt từ người duy trì.
  (All PRs require at least one approval from a maintainer.)
- Các kiểm tra CI phải vượt qua.
  (Continuous Integration CI checks must pass.)
- Chúng tôi có thể yêu cầu thay đổi hoặc làm rõ trước khi merge.
  (We may ask for changes or clarifications before merging.)

Cảm ơn sự hỗ trợ của bạn! / Thank you for your help!
